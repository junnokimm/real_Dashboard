Variant B 만들기

바꾸고 싶은 요소를 클릭한 뒤, 아래에서 원하는 작업을 고르세요. 기술 용어는 숨기고, 자주 쓰는 실험만 바로 만들 수 있게 구성했어요.
버튼, 제목, 설명 문구처럼 텍스트가 중심인 요소에 적합해요.
위치 이동은 안전하게 `transform`으로 처리돼요. 아주 큰 이동보다는 살짝 강조하거나 정렬을 다듬는 데 쓰는 걸 권장해요.
특정 안내문, 배지, 배너처럼 노출 여부가 중요한 요소에 적합해요.
링크나 버튼 클릭 후 이동하는 주소를 바꾸고 싶을 때 사용해요.
고급 편집 열기
- 기술 Action, selector, raw CSS는 디버그/예외 처리용이에요.
- 일반적인 운영은 위의 쉬운 편집 모드만으로도 충분하게 설계하는 게 목표예요.
- 프리뷰 적용: iframe에서만 임시로 보임(저장 안 됨)
- Real 적용: 서버에 저장/배포 → 실제 방문 시 SDK가 자동 A/B 실행
- 목표는 관리자가 코드 없이도 실험을 만들 수 있게 하는 것이고, 어려운 기능은 고급 편집 안으로 숨겨둘 거예요.
